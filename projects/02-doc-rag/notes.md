# 架构设计决策

## 1. 本地 JSON 存储 vs 向量数据库

**决策**：使用本地 JSON 文件（`.data/vectors.json`）存储向量。

**理由**：

- 学习项目数据量小（< 100 篇文档），JSON 足够胜任
- 零依赖，无需安装配置 Pinecone/Milvus/Chroma 等向量数据库
- 可读性好，方便调试和理解向量检索原理
- 瓶颈在 Embedding API 调用而非检索速度，本地文件 IO 不是瓶颈

**未来升级**：数据量 > 1000 篇时迁移到 Chroma 或 LanceDB。

## 2. 固定大小切分 vs 按标题切分

**决策**：v1/v2 使用固定 500 字切分 + 100 字重叠。

**理由**：

- 实现简单可靠，不会因 Markdown 格式不规范而失效
- 重叠窗口保证跨 chunk 边界的语义不被切断
- Week 5 计划对比按标题切分效果，用数据驱动优化

## 3. OpenRouter 作为 LLM 网关

**决策**：默认使用 OpenRouter 的免费模型 `openai/gpt-oss-120b:free`。

**理由**：

- 免费额度足够开发测试，降低学习成本
- 通过 `OPENAI_BASE_URL` 可一键切换为 OpenAI / DeepSeek / 本地模型
- HTTP-Referer 和 X-Title 头满足 OpenRouter 使用规范

## 4. Express + 静态 HTML vs 全栈框架

**决策**：使用 Express 做后端 API，纯静态 HTML/CSS/JS 做前端。

**理由**：

- 聚焦 RAG 核心能力，不引入 React/Vue 框架噪声
- 单文件 HTML 即可完成上传 + 问答 + 来源展示
- 后端 API 独立可测，方便后续替换前端或接入其他客户端

## 5. 安全加固策略

**决策**：实施三层防护。

| 层级     | 措施                                | 原因                                    |
| -------- | ----------------------------------- | --------------------------------------- |
| 路径穿越 | `basename()` + `resolve()` 二次校验 | 防止 `../../../etc/passwd` 覆盖系统文件 |
| 文件大小 | multer 限制 10MB                    | 防止单个大文件耗尽内存/OOM              |
| 文件类型 | 仅允许 `.md`/`.txt`（含大写扩展名） | 防止上传可执行文件或非文本内容          |

multer 从 1.x 升级到 2.1.1 修复了已知安全漏洞，且 API 变动较小（`req.files` 类型声明需调整）。

## 6. .env 加载策略

**决策**：根目录 `.env` 优先，项目级 `.env` 作为 fallback（`override: false`）。

**理由**：

- 多个学习项目共享同一套 API Key，统一管理避免重复配置
- 项目特定变量（如 `RAG_PORT`）可在项目级 `.env` 覆盖
- `override: false` 保证已设置的环境变量不会被 dotenv 覆盖，支持 CI/CD 注入

## 7. 空知识库处理

**决策**：索引时若知识库目录为空，清空 `vectors.json` 写 `[]`。

**理由**：

- 防止用户删除所有文档后，旧向量残留导致问答返回"伪有效"结果
- 保持文件系统状态与向量索引一致性
