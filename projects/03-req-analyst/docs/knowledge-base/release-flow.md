# 发布上线流程

## 发布环境

| 环境    | 域名                | 用途         | 自动构建        |
| ------- | ------------------- | ------------ | --------------- |
| dev     | dev.example.com     | 开发联调     | 否              |
| staging | staging.example.com | 测试/QA 验收 | 否              |
| pre     | pre.example.com     | 预发布验证   | 是（main 分支） |
| prod    | example.com         | 正式环境     | 手动触发        |

## 发布流程

```
开发自测 → 提交 CR → 合并 main
  → pre 环境自动部署
  → QA 在 pre 环境回归
  → 产品验收
  → 发布 prod（手动触发）
  → 线上监控 30 分钟
```

## 灰度发布

生产环境使用 CDN 灰度策略：

| 阶段 | 流量比例 | 观察时间 | 回滚条件       |
| ---- | -------- | -------- | -------------- |
| 1    | 5%       | 10 分钟  | 错误率 > 1%    |
| 2    | 30%      | 15 分钟  | 错误率 > 0.5%  |
| 3    | 100%     | -        | 全量后持续观察 |

## 发布检查清单

- [ ] 前端埋点已注册（pageView + click + biz_event）
- [ ] 新增接口已配置网关路由 + 鉴权
- [ ] 敏感操作（支付、删除）有二次确认
- [ ] 异常状态有兜底 UI（网络错误、空数据、接口超时）
- [ ] 图片/资源使用 CDN 域名，非本地路径
- [ ] 涉及支付的页面走 HTTPS，URL 不传 token

## 回滚方案

- pre 环境保留最近 3 个版本的构建产物
- prod 环境保留最近 5 个版本
- 回滚操作：运维在发布平台选择目标版本 → 一键回滚，CDN 缓存 5 分钟内生效
- 数据库变更（migration）不支持自动回滚，需手动执行回滚脚本

## 监控与告警

| 指标         | 阈值   | 通知渠道          | 处理时效  |
| ------------ | ------ | ----------------- | --------- |
| JS 错误率    | > 0.5% | 企业微信群 + 邮件 | 15 分钟内 |
| 首屏加载时间 | > 3s   | 邮件              | 1 小时内  |
| API 超时率   | > 1%   | 邮件              | 30 分钟内 |
| 构建失败     | 每次   | 企业微信 @提交人  | 及时修复  |
