# 阶段 03：Agent

## 这一阶段在学什么

这一阶段对应 Week 7–8，重点是把系统从“知道什么”推进到“先做什么”：

- Tool Calling
- ReAct 循环
- 多步任务拆解
- 工具注册与回填
- 安全边界和只读约束

## 为什么 Agent 不是前置，而是后置

Agent 的价值建立在前面两层之上：

- 没有稳定的 Prompt / API 基本功，工具调用会很脆
- 没有 RAG / 检索底座，Agent 只能空转

所以 Agent 不该最先学，而应该在前面知识和工程边界都看清以后再学。

## 这一阶段真正学会了什么

- Agent 的核心不在“会不会调工具”，而在“工具边界清不清楚”
- 真正难的是迭代上限、结果截断、路径安全、大文件策略、缓存失效这些工程约束
- 最终项目更像前面能力的汇合点，而不是孤立的新产品

## 这一阶段的完成标志

- 你已经能解释 RAG 和 Agent 的分工
- 你已经能说明为什么只读比直接写代码更适合作为第一版
- 你已经能把“项目、概念、阶段、判断”收成一套一致叙事
