# 阶段 01：Foundation

## 这一阶段在学什么

这一阶段对应 Week 1–2，重点不是“做功能”，而是建立最小 AI 应用工程闭环：

- Prompt 是怎么控制输出的
- LLM API 怎么接
- streaming、token、错误处理这些工程问题怎么处理

## 为什么从这里开始

如果一开始就上 RAG 或 Agent，很容易变成“会拼概念，不会落系统”。

所以第一阶段先解决一个更基本的问题：

怎么让模型先稳定地进入一个真实可运行的程序。

## 这一阶段的代表产物

- `projects/01-ai-code-explain/`

它不是最终产品，而是第一个“能跑、能测、能解释真实文件”的最小闭环。

## 这一阶段真正学会了什么

- Prompt 不是描述需求那么简单，它本质上是在设计输出边界
- LLM API 的可用性，不只取决于模型，还取决于上下文和错误处理
- 一个最小 CLI demo 的价值，在于把数据流和系统边界看清楚

## 为什么会进入下一阶段

当文件和文档变长之后，单次 prompt 迟早会撞上上下文上限。

这时就必须进入 Embedding 和检索，也就是 RAG 阶段。
