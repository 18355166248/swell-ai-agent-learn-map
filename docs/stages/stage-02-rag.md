# 阶段 02：RAG

## 这一阶段在学什么

这一阶段对应 Week 3–6，重点是把“模型知道什么”这件事做成可控系统：

- Embedding
- Chunk
- 检索
- 引用来源
- 检索优化
- 真实文档接入

## 为什么这一阶段重要

很多 AI 应用看起来是在“提问”，本质上其实是在“找信息”。

如果检索质量不稳定，后面的生成和分析都会漂。

所以这一阶段不是一个附属能力，而是 AI Native 系统的知识底座。

## 这一阶段的代表产物

- `projects/02-doc-rag/`
- `projects/03-req-analyst/`
- `experiments/chunk-strategies/`

## 这一阶段真正学会了什么

- RAG 的难点不是接一个 embedding API，而是“怎么切、怎么存、怎么查、怎么验证”
- 检索质量不能靠感觉调，必须靠实验
- 真实场景接入后，RAG 才会从“知识问答”变成“研发辅助”

## 为什么会进入下一阶段

当问题从“告诉我答案”变成“帮我完成一个分析任务”时，单次检索已经不够了。

系统需要自己决定先列目录、再搜代码、再读文档，这就进入 Agent 阶段。
