# 阶段总结索引

> 这里不是按“项目”看仓库，而是按“成长阶段”看仓库。每一阶段都回答同一组问题：为什么学、学会了什么、做成了什么、为什么会进入下一阶段。

## 阶段列表

| 文件                     | 阶段                            | 核心问题                                 |
| ------------------------ | ------------------------------- | ---------------------------------------- |
| `stage-01-foundation.md` | Prompt / API / 工程基本功       | 怎么把模型稳定接进一个最小系统？         |
| `stage-02-rag.md`        | Embedding / RAG / 检索优化      | 怎么让系统不靠硬塞上下文，而是会找知识？ |
| `stage-03-agent.md`      | Tool Calling / ReAct / 工具边界 | 怎么让模型从“知道”变成“去做”？           |
| `stage-04-next.md`       | Evaluation / Memory / Workflow  | 下一阶段应该往哪里继续扩？               |

## 怎么使用这组文档

- 想看“每周做了什么”：看 `weeks/`
- 想看“每个阶段产出了什么”：看 `projects/`
- 想看“方法怎么验证”：看 `experiments/`
- 想看“我到底在每一阶段学会了什么”：看这里

## 这层目录存在的意义

如果没有这层阶段总结，仓库很容易被看成“4 个小项目 + 一堆周记”。

有了这层之后，整个仓库会更清楚地表达出：

- 你不是在堆 demo
- 你是在按阶段学习 AI Native / AI Agent
- 每个项目都只是阶段性载体，不是唯一目标
