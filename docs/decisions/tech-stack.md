## TDR-001：存储方案选择

**时间**：Week 3-4

**决策**：第一版使用本地 JSON 文件存储向量，而不是向量数据库。

**原因**：

- 降低上手门槛，专注于理解 RAG 核心逻辑
- 文档量小时（< 1000 chunks）性能足够
- 后期可以无缝替换为 pgvector

**替换时机**：文档量超过 1000 chunks 或需要多人共享时。

## TDR-002：技术栈选择

**时间**：Week 1

**决策**：TypeScript + Node.js + OpenAI SDK，不用 LangChain。

**原因**：

- 直接用 SDK 能真正理解底层机制
- LangChain 封装太多，适合熟悉原理后再用
- 前端工程师用 TS 最顺手
