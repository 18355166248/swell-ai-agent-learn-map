## 要研究的问题

- [ ] 为什么搜不到（chunk 太大 / 太小 / 语义割裂）
- [ ] 为什么搜错了（Query 和文档表达方式不一样）
- [ ] 回答太泛（Top-K chunk 太多，噪声太大）
- [ ] 怎么量化效果（建评估数据集）

## 三个优化方向

### 1. Chunk 策略优化

| 策略              | 适合场景   | 缺点       |
| ----------------- | ---------- | ---------- |
| 固定 500 字       | 快速实现   | 切断语义   |
| 按 Markdown 标题  | 文档类内容 | 块大小不均 |
| 按代码块/接口定义 | 技术文档   | 需要解析   |

实验记录见：`../../experiments/chunk-strategies/`

### 2. Query 改写

用户问："图片怎么上传？"
改写后："图片压缩、上传 CDN、替换本地资源路径、上传 SDK 使用流程"

### 3. Hybrid Search（混合检索）

```
向量检索（语义相近）+ BM25 关键词检索（精确词匹配）→ 结果合并
```

工程文档中有大量关键词（`uploadImage`、`cdnUrl`、`activityId`），只靠向量搜索会漏掉。
