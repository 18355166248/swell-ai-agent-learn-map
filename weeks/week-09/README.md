### 学到的东西

1. **评估要在调 Prompt 之前** — 先有固定任务集和评分标准，调参才有方向
2. **token 限制是隐藏的评估障碍** — Round 7 agent-002 失分只是因为 2048 token 截断，不是能力问题
3. **安全约束验证必须独立于功能验证** — constraint_ok 和 task_completed 是两个正交维度
4. **回归对比让进展可视化** — `passRateDelta` 一眼看出是变好还是变差
