# Week 9：Phase 2A · Evaluation

**时间**：下一阶段启动后填写  
**状态**：⬜ 未开始

## 本周目标

> 从“做出了 RAG / Agent 原型”进入“如何证明它真的变好了”。这一周的核心不是新功能，而是建立评估思维。

## 这一周要解决的问题

- 怎么定义一个 RAG 任务是否成功
- 怎么定义一个 Agent 任务是否完成
- 怎么记录失败类型，而不是只看最终答案
- 怎么把“测试全绿”和“效果真的更好”区分开

## 本周交付重点

- `projects/05-agent-eval/` 项目骨架
- `experiments/agent-evals/` 评估任务集模板
- 第一版 RAG / Agent 评估维度设计
- 下一阶段需要长期维护的回归检查思路

## 建议学习顺序

1. 先定义任务集
2. 再定义评分维度
3. 再定义失败分类
4. 最后考虑评估执行脚本
