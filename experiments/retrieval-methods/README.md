# 检索方法对比实验

## 实验目的

对比不同检索方法在工程文档场景下的效果，为 `projects/02-doc-rag/` 选择最优方案。

## 方法对比

### 方法 A：纯向量检索

**原理**：把问题和文档都转成向量，计算余弦相似度，返回最相似的 K 个 chunk。

**优点**：能捕捉语义相近的表达（"上传图片" vs "图片传 CDN"）。

**缺点**：对精确关键词不敏感（`uploadImage`、`activityId` 这类词的向量相似度不一定高）。

### 方法 B：BM25 关键词检索

**原理**：基于词频/逆文档频率的传统搜索算法，对精确词匹配非常敏感。

**优点**：函数名、配置项、枚举值等精确词匹配效果极好。

**缺点**：无法处理同义词和语义相近的表达。

### 方法 C：混合检索（Hybrid Search）

**原理**：向量检索 + BM25 各跑一次，用 RRF（Reciprocal Rank Fusion）合并结果排序。

**优点**：兼顾语义理解和精确词匹配，工程文档场景综合效果最好。

**缺点**：实现复杂度高一些，需要维护两套索引。
